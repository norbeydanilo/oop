# Encapsulamiento en Java

## ¿Qué es el encapsulamiento?

El encapsulamiento consiste en **proteger los datos de una clase** para que no puedan ser modificados directamente desde afuera. En cambio, se controla el acceso a esos datos a través de métodos especiales.

La idea viene del mundo real: una caja fuerte guarda algo valioso y solo permite interactuar con su contenido de formas controladas (abrir con clave, ver el saldo, depositar). Nadie puede meter la mano directamente.

---

## ¿Por qué es importante?

Sin encapsulamiento, cualquier parte del programa podría cambiar los datos de una clase de forma incorrecta. Por ejemplo, asignarle una edad negativa a una persona, o dejar el nombre vacío. El encapsulamiento permite **validar y controlar** qué valores se aceptan.

---

## Las dos reglas del encapsulamiento

1. Los atributos se declaran como `private` — solo la propia clase puede acceder a ellos directamente.
2. Se crean métodos `public` para leer y modificar esos atributos — estos métodos se llaman **getters** y **setters**.

| Método | Para qué sirve | Ejemplo |
|---|---|---|
| **getter** | Leer el valor de un atributo | `getNombre()` |
| **setter** | Modificar el valor de un atributo | `setNombre(...)` |

---

## Código explicativo

```java
// Archivo: Persona.java
// Esta clase representa a una persona con nombre y edad.
// Usamos encapsulamiento para proteger sus datos.

public class Persona {

    // Los atributos son PRIVATE: solo esta clase puede tocarlos directamente.
    // Desde afuera, nadie puede escribir: persona.nombre = "algo" (eso daría error).
    private String nombre;
    private int edad;


    // --- GETTERS ---
    // Un getter es un método que DEVUELVE el valor de un atributo.
    // Es public para que cualquiera pueda leerlo, pero no modificarlo directamente.

    public String getNombre() {
        return nombre;
    }

    public int getEdad() {
        return edad;
    }


    // --- SETTERS ---
    // Un setter es un método que MODIFICA el valor de un atributo.
    // Aquí podemos poner reglas: por ejemplo, no aceptar edades negativas.

    public void setNombre(String nuevoNombre) {
        nombre = nuevoNombre;
    }

    public void setEdad(int nuevaEdad) {
        // Validación: solo aceptamos la edad si es un número positivo
        if (nuevaEdad > 0) {
            edad = nuevaEdad;
        } else {
            System.out.println("Error: la edad no puede ser negativa.");
        }
    }

}
```

```java
// Archivo: App.java
// Aquí probamos la clase Persona y vemos el encapsulamiento en acción.

public class App {

    public static void main(String[] args) {

        // Creamos un objeto de tipo Persona
        Persona p = new Persona();

        // Usamos los SETTERS para asignar valores
        p.setNombre("Carlos Ruiz");
        p.setEdad(20);

        // Usamos los GETTERS para leer los valores
        System.out.println("Nombre: " + p.getNombre());
        System.out.println("Edad: " + p.getEdad());

        // Intentamos asignar una edad inválida — el setter nos lo impide
        p.setEdad(-5);

        // La edad no cambió porque el setter la rechazó
        System.out.println("Edad después del intento inválido: " + p.getEdad());

    }

}
```

### Salida esperada en consola

```
Nombre: Carlos Ruiz
Edad: 20
Error: la edad no puede ser negativa.
Edad después del intento inválido: 20
```

---

## Lo que NO se debe hacer

```java
// Esto está MAL — acceso directo al atributo privado (Java no lo permitirá)
p.edad = -5;       // ❌ Error de compilación
p.nombre = "Ana";  // ❌ Error de compilación

// Esto está BIEN — se accede a través de los métodos
p.setEdad(25);          // ✅
p.setNombre("Ana");     // ✅
```

---

## Resumen

| Concepto | Significado |
|---|---|
| `private` | El atributo solo es accesible dentro de su propia clase |
| `public` | El método es accesible desde cualquier parte del programa |
| getter | Método que **retorna** el valor de un atributo |
| setter | Método que **modifica** el valor de un atributo, con posibilidad de validar |

> El encapsulamiento no es solo una regla de sintaxis — es una decisión de diseño. Decide qué información de tu clase es visible hacia afuera y qué no.
