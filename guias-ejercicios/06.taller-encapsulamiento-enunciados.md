# Taller: Encapsulamiento en Java

Practica el encapsulamiento creando clases con atributos privados, getters, setters y métodos.

---

## Ejercicio 1 — Libro

Crea una clase llamada `Libro` que represente un libro de una biblioteca. La clase debe tener los siguientes **atributos privados**:

- `titulo` (String) — el título del libro
- `autor` (String) — el nombre del autor
- `disponible` (boolean) — indica si el libro está disponible para préstamo

La clase debe tener:
- Un **getter y setter** para cada atributo.
- El setter de `titulo` debe rechazar títulos vacíos.
- Un método `mostrarInfo()` que imprima en consola los datos del libro.
- Un método `prestar()` que marque el libro como no disponible e imprima un mensaje. Si ya estaba prestado, debe avisarlo.
- Un método `devolver()` que marque el libro como disponible e imprima un mensaje.

En `App.java` crea **dos objetos** de tipo `Libro`, asigna sus datos y prueba todos los métodos.

## Ejercicio 2 — Vuelo

Crea una clase llamada `Vuelo` que represente un vuelo aéreo. La clase debe tener los siguientes **atributos privados**:

- `numero` (String) — código del vuelo, por ejemplo `"AV9401"`
- `origen` (String) — ciudad de origen
- `destino` (String) — ciudad de destino
- `ocupacion` (int) — número de pasajeros a bordo
- `capacidadMaxima` (int) — máximo de pasajeros permitidos

La clase debe tener:
- Un **getter y setter** para cada atributo.
- El setter de `ocupacion` debe rechazar valores negativos o mayores a `capacidadMaxima`.
- Un método `mostrarInfo()` que imprima los datos del vuelo.
- Un método `embarcar(int pasajeros)` que sume pasajeros a la ocupación. Si no hay suficiente espacio, debe avisarlo sin cambiar la ocupación.
- Un método `desembarcar(int pasajeros)` que reste pasajeros. No puede quedar en negativo.

En `App.java` crea **dos objetos** de tipo `Vuelo` y prueba todos los métodos.

## Ejercicio 3 — Depósito de agua

Observa el siguiente diagrama y crea la clase correspondiente:

```
┌─────────────────────┐                    ┌─────────────────────┐
│   Depósito de agua  │ ── desbordamiento ─→│   Depósito de agua  │
├─────────────────────┤                    ├─────────────────────┤
│ capacidad           │                    │ capacidad           │
│ volumenActual       │                    │ volumenActual       │
│ depositoDesborde    │ ← relación con     ├─────────────────────┤
├─────────────────────┤   otro depósito    │ agregarAgua(cant)   │
│ agregarAgua(cant)   │                    │ quitarAgua(cant)    │
│ quitarAgua(cant)    │                    └─────────────────────┘
└─────────────────────┘
```

Cuando el depósito principal se desborda, el agua sobrante **no se pierde**: va a llenar otro depósito de la misma clase.

La clase `DepositoAgua` debe tener:
- Atributos privados `capacidad` y `volumenActual` (ambos `double`).
- Un atributo privado `depositoDesborde` de tipo `DepositoAgua` — representa el depósito que recibirá el agua sobrante. Puede ser `null` si no hay ninguno conectado.
- Getters y setters para todos los atributos. El setter de `capacidad` debe rechazar valores menores o iguales a cero.
- Un método `mostrarEstado()` que imprima la capacidad, el volumen actual y el espacio libre.
- Un método `agregarAgua(double cantidad)` que sume al volumen. Si se supera la capacidad, el volumen queda en el máximo y el agua sobrante se envía al `depositoDesborde` llamando a su propio método `agregarAgua()`. Si no hay depósito de desborde, solo se imprime un aviso.
- Un método `quitarAgua(double cantidad)` que reste del volumen. Si no hay suficiente agua, el volumen queda en cero y se imprime un aviso.

En `App.java` crea **dos objetos**: conéctalos de modo que el desbordamiento del primero llene el segundo, y prueba el caso de desbordamiento.
